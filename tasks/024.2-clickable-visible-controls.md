# Task 024.2: Clickable Visible Controls

## Goal

Make the visible controls respond to mouse clicks in the running Ebitengine app.

## Scope

- Poll mouse input during `Update`.
- Hit-test clicks against the visible control rectangles.
- Clicking mode controls changes the active editor mode.
- Clicking command controls runs the same command as the matching keyboard shortcut.
- Clicking Pause pauses the simulation and clicking Run resumes it.
- Clicking Reset resets the world.
- Clicking Load, Insert, and Save opens keyboard path entry for that command.
- Clicking Quit requests app close.
- Clicked controls visibly update active state on the next frame.
- Clicks outside controls do not trigger control commands.

## Acceptance Notes

- Rendered labels alone are not sufficient.
- The same command path should be used for mouse clicks and keyboard shortcuts.
- File command path entry can be simple keyboard text entry; native dialogs are not required.

## Done When

- A user can operate visible mode and command controls with the mouse.
- Acceptance and focused app tests cover control hit-testing and command dispatch.
