# Task 024.1: Render Visible Controls

## Goal

Render the toolbar, command bar, inspector, and status line visibly in the actual Ebitengine window.

## Scope

- Draw the left toolbar into the `Draw` output.
- Draw the top command bar into the `Draw` output.
- Draw the right inspector into the `Draw` output.
- Draw the bottom status line into the `Draw` output.
- Draw readable labels for visible controls and status fields.
- Draw visible panel boundaries or contrasting backgrounds so each region is distinguishable from the canvas.
- Keep world content visible in the remaining canvas area.
- Add a render-level check that the UI chrome produces non-background pixels outside the canvas.

## Acceptance Notes

- Model-only screen/control data is not enough for this task.
- The visible controls can be simple custom-drawn rectangles and text, but they must appear in the real app window.
- TPS/debug text must not count as toolbar, command bar, inspector, or status line rendering.
- Text must fit within the visible control regions at the default window size.

## Done When

- Running the app shows the left toolbar, top command bar, right inspector, and bottom status line without user action.
- A visual or pixel-level test proves those regions are not blank/background-only.
- The startup demo world remains visible alongside the UI chrome.
